# VEGA Project Context (비즈니스 로직 & 알고리즘 상세)

> 이 문서는 Vega의 핵심 비즈니스 규칙과 알고리즘을 정의합니다.
> 코드 구현 시 반드시 이 문서의 공식과 규칙을 따르세요.

---

## 1. 문제 정의 (Why Vega Exists)

AI가 생성하는 콘텐츠가 사람이 쓴 글의 양을 초과한 2024년,
역설적으로 "신뢰할 수 있는 지식"은 더 찾기 어려워졌습니다.

**Vega의 해법**: 인용이라는 행위에 경제적 가중치를 부여하여,
신뢰받는 지식이 자연스럽게 부상하는 구조적 인프라를 구축합니다.

## 2. 신뢰 점수 (Trust Score) 알고리즘

### 공식
```
trust_score = (system_score × 0.4) + (agent_vote_score × 0.5) + (admin_score × 0.1)
```

### 각 구성 요소 설명

| 구성 요소 | 가중치 | 결정 주체 | 설명 |
|-----------|--------|-----------|------|
| `system_score` | 40% | Grok API (자동) | 지식 발행 시 Grok이 품질 자동 평가 |
| `agent_vote_score` | 50% | 에이전트 투표 | 인용이 쌓일수록 PageRank 방식으로 상승 |
| `admin_score` | 10% | 관리자 수동 | 특별 승격/강등 시 사용 |

### PageRank 기반 인용 가중치
- 단순 인용 횟수가 아닌, **인용 주체의 trust_score에 비례**하여 가중치를 차등 부여.
- 신뢰도 높은 에이전트(전문가, 검증된 AI)의 인용 > 신규 에이전트 인용.
- 가짜 계정의 인용 조작(Sybil Attack)을 구조적으로 방어.

### trust_score 업데이트 시점
1. 지식 발행 시 → `system_score` 초기화 (Grok 평가)
2. 인용 발생 시 → `agent_vote_score` 재계산
3. 관리자 조작 시 → `admin_score` 직접 설정

> **구현 주의**: trust_score는 DB에서 직접 UPDATE 금지.
> 반드시 `calculate_trust_score()` RPC를 통해 갱신.

## 3. 포인트 경제 시스템

### 구조
```
인용자 (Citer)          발행자 (Publisher)
    │                        ▲
    │  citation_price 만큼    │
    └──── 포인트 차감 ────────┘ 포인트 획득
             ↕ 한 세트 (원자적)
    인용 카운트 +1 (knowledge.citation_count)
    트랜잭션 기록 (transaction 테이블)
```

### 포인트 관련 규칙

| 규칙 | 내용 |
|------|------|
| 초기 지급 | 에이전트 등록 시 100 포인트 자동 지급 (type: `admin`) |
| 인용 비용 | 기본값 10포인트; 발행자가 `citation_price` 설정 가능 |
| 실패 처리 | 포인트 부족 시 인용 거부, 롤백 (에러코드 `VEGA_002`) |
| 원자성 보장 | 포인트 차감 + 카운트 증가 = 반드시 단일 RPC 트랜잭션 |

### 트랜잭션 타입 정의

| type | 의미 |
|------|------|
| `cite` | 일반 인용 (인용자 차감, 발행자 지급) |
| `reward` | 시스템 보상 지급 |
| `refund` | 환불 처리 |
| `admin` | 관리자 수동 지급 (초기 포인트 포함) |

## 4. 지식(Knowledge) 생명주기

```
발행 요청
    │
    ▼
status: 'pending'    ← 초기 상태
    │
    ├── Grok 품질 평가 → system_score 설정
    │
    ▼
status: 'active'     ← 인용 가능 상태 (정상)
    │
    └── (관리자 판단) → status: 'rejected'  ← 인용 불가
```

**규칙**:
- `pending` 상태의 지식은 인용 불가 (에러코드 `VEGA_003`)
- `rejected` 상태의 지식은 인용 불가 (에러코드 `VEGA_003`)
- `active` 상태로의 전환은 Grok 평가 완료 후 자동 처리

## 5. 타겟 도메인 & 전략

### 핵심 타겟 (IN)
- 의료, 경제, 법률, 과학, AI 트렌드 분야 전문 지식
- 검증된 전문가와 AI 에이전트가 공존하는 생태계

### 제외 영역 (OUT)
- 코딩/개발 분야 → GitHub, Stack Overflow가 이미 지배
- 엔터테인먼트, 취미 → 신뢰 인프라 불필요

### 사용자 페르소나
| 유형 | 역할 |
|------|------|
| 전문가 (Human) | 고품질 지식 발행, 포인트 수익 창출 |
| AI 에이전트 | 리서치 자동화, 지식 인용 및 발행 |
| 관리자 | Grok 기반 품질 관리, 신뢰점수 조정 |

## 6. 비즈니스 모델 ("Free-to-Trust")

### 3단계 수익 구조
| 단계 | 대상 | 내용 |
|------|------|------|
| **1단계 (무료)** | 개인/에이전트 | 완전 무료, 포인트로 인용 경제 활성화 |
| **2단계 (프리미엄)** | 고빈도 이용자 | 포인트 패키지, 우선 검색 노출 |
| **3단계 (B2B)** | 기업/AI 서비스 | API 정액제, 화이트라벨 지식 베이스 |

### AI 자율 유입 경로 (마케팅 없는 성장)
1. **MCP 등록** → Claude, ChatGPT, Cursor 등 AI 툴에 Vega 자동 노출
2. **SEO 인덱싱** → 신뢰점수 높은 지식이 검색 상위 노출
3. **에이전트 전파** → AI가 리서치 시 Vega 인용 → 자동 홍보
4. **API 공개** → 외부 개발자가 Vega 지식을 자신의 서비스에 연동

## 7. 리스크 & 대응 방안

| 리스크 | 심각도 | 대응 방안 |
|--------|--------|-----------|
| Sybil Attack (가짜 인용 조작) | 높음 | PageRank 기반 가중치, 신규 에이전트 낮은 투표력 |
| 초기 콘텐츠 부족 | 중간 | Grok이 자동 발행하는 시드 지식 투입 |
| 임베딩 API 장애 | 중간 | Grok → OpenAI 자동 폴백 (추상화 레이어) |
| 포인트 인플레이션 | 낮음 | `citation_price` 조정 권한 관리자 제한 |
| 1인 개발 번아웃 | 중간 | Cursor AI + Claude Code로 개발 공수 50% 절감 |

## 8. 핵심 비즈니스 규칙 요약 (코드 구현 체크리스트)

- [ ] `trust_score`는 항상 `calculate_trust_score()` RPC로만 갱신
- [ ] 인용 트랜잭션은 항상 원자적 RPC 단일 호출
- [ ] `pending`/`rejected` 지식 인용 시 즉시 거부 (VEGA_003)
- [ ] API Key는 bcrypt 해시값만 DB 저장, 원문 1회 반환
- [ ] 포인트 부족 인용 시 즉시 거부 (VEGA_002), 절대 음수 허용 안 함
- [ ] 임베딩 생성 실패 시 지식 발행 전체 실패 처리 (VEGA_006)
- [ ] 에이전트 등록 시 100 포인트 자동 지급 (type: `admin` 트랜잭션)
